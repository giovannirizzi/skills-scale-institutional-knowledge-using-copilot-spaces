# OctoAcme Project Management Docs

This README provides an overview of the project management processes used by OctoAcme and links to detailed documentation for each major process area. Use this as the starting point to understand how OctoAcme runs, plans, executes, and improves projects.

## Purpose of This Space

This Copilot Space centralizes OctoAcme's project management knowledge so that it is:

- **Searchable and standardized** – all process documentation lives in one place, written in a consistent format.
- **Onboarding acceleration** – new teammates can quickly understand OctoAcme's approach, roles, and key artifacts without needing to ask around.
- **Process improvement feedback loops** – a single source of truth makes it easy to identify gaps, propose updates, and track improvements over time.

## Project Management Process Overview

OctoAcme follows a stage-based, customer-first project lifecycle:

- **Principles**: customer-first, iterative delivery, clear ownership, data-informed decisions, and psychological safety.
- **Core roles**: Project Manager (PM), Product Manager (PdM), Developers, Designer, QA/Test Engineer, Data Analyst, Security/Compliance Stakeholder, Support/Customer Success, and Executive Sponsor.
- **Lifecycle stages**: Initiation → Planning → Execution → Release → Close & Retrospective.
- **Key artifacts**: Project Charter, Roadmap and Release Plan, Sprint Backlog, Acceptance Criteria & Definition of Done, Risk Register, and Retrospective notes.
- **Communication cadence**: weekly PM + PdM sync, twice-weekly standups, monthly stakeholder updates, and ad-hoc escalations.
